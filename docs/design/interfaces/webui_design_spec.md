# ShinBot 技术规范：WebUI (WebUI Spec)

本文档作为 Dashboard 的单一主规范，统一描述前端工程基座、页面结构、组件模式、i18n 约束与协作规则。

---

## 1. 工程基座

- **框架**: Vue 3 (Composition API)
- **构建**: Vite + pnpm
- **UI**: Vuetify 3
- **状态管理**: Pinia
- **国际化**: Vue I18n
- **通信**: Axios + WebSocket

所有业务路由必须经过鉴权守卫；未登录用户只能访问 `/login`。

---

## 2. 布局与导航

### 2.1 主布局

- **Sidebar**: 支持折叠，承载主导航。
- **Header**: 展示当前系统状态、通知和用户菜单。
- **Main**: 承载页面内容区域。

### 2.2 核心页面

- **Welcome**: 首次配置引导。
- **Dashboard**: 概览指标与告警摘要。
- **Instances**: 机器人实例管理。
- **Plugins**: 插件管理与配置。
- **Permissions**: 权限组与规则管理。
- **Monitoring**: 日志、运行状态与后续审计入口。

---

## 3. 页面模板

所有列表/管理类页面遵循统一结构：

1. `Page Header`: 标题、描述、全局操作。
2. `Control Bar`: 视图切换、搜索、筛选、排序。
3. `Main Content`: 卡片网格或列表表格。
4. `Pagination`: 仅在确有分页时显示。

展示类页面应同时支持：

- **Card View**: 快速概览和轻操作。
- **List View**: 批量操作和详细对比。

---

## 4. 组件模式

### 4.1 卡片

卡片是实例、插件、权限组等实体的默认展示单元。

- **Header**: 图标/头像、标题、状态、主操作。
- **Body**: 2-3 行关键信息摘要。
- **Footer**: 编辑、配置、删除等操作入口。

推荐网格断点：

- `xs=12`
- `sm=6`
- `md=4`
- `lg=3`

### 4.2 过滤条

统一使用独立过滤条组件承载：

- 搜索
- 状态筛选
- 分类筛选
- 排序

### 4.3 叠层与反馈

- **Drawer**: 轻量级详情或侧边设置。
- **Dialog**: 关键修改和危险操作确认。
- **Snackbar/Toast**: 所有操作结果反馈。
- **Skeleton**: 优先于全屏 loading。

---

## 5. i18n 约束

i18n 文件按业务模块放在 `src/locales/{lang}/` 下，并遵循“目录即命名空间”。

推荐结构：

```text
src/locales/zh-CN/
├── common/
├── layout/
└── pages/
```

约束：

- 禁止在 `.vue` 中硬编码自然语言文本。
- 新增键值必须跨语种同步。
- JSON 内部最多保留一层逻辑嵌套。
- 动态文本使用占位符，不手拼字符串。

更详细的目录与键名规范见 `docs/dashboard/i18n/structure.md`。

---

## 6. 开发协作约束

- 表单提交按钮必须显式绑定 `loading` 状态。
- 危险操作必须二次确认。
- Pinia 中的状态切分按业务域进行，不做“全局超级 store”。
- API 成功与失败都必须给出可见反馈。
- 页面设计优先保持一致性，其次再做局部差异化。
