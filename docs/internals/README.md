# Internals Docs

`docs/internals/` 记录当前代码实现中的关键机制。这里的文档回答“现在代码实际如何工作”，不是长期架构约束，也不是用户配置指南。

## 当前文档

- `workflow_engine.md`
  - 消息入口、路由表、Agent fallback 边界和工作流交付路径。
- `plugin_lifecycle.md`
  - 插件加载、启用、禁用、卸载期间的注册与清理行为。
- `business_logic_infra.md`
  - 业务逻辑基础设施的当前实现说明。
- `adapter_abstraction.md`
  - 适配器抽象层与平台接入实现说明。
- `ast_and_serialization.md`
  - 消息 AST 与序列化实现说明。
- `parameters/`
  - 当前仍影响行为或性能、但暂不作为普通用户配置暴露的内部参数。

## 放置规则

- 与源码结构强绑定、会帮助后续维护的实现说明放在这里。
- 跨模块职责边界放在 `../architecture/`。
- 子系统能力规格放在 `../design/`。
- 插件作者需要稳定依赖的接口放在 `../plugins/`。
- 一次性排障记录、阶段总结、迁移计划不放在这里。

## 维护规则

- 文档应描述当前实现，避免保留已过期的计划语气。
- 如果实现说明被新的设计文档取代，应移入 `../archive/` 并在顶部标明替代文档。
- 涉及参数默认值、阈值和状态机条件时，优先给出对应代码位置。
