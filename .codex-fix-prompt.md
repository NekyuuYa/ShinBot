Fix 3 runtime-breaking bugs in the Pydantic response models on this branch. Do NOT change any route logic — only fix the models and their declarations.

## Bug 1 (P1) — config_providers.py: ConfigProviderSummary strips provider fields

The `ConfigProviderSummary` model at line ~27 only has `kind`, `id`, `name`, `schemaRef`, `defaultsRef`, `validateRef`, `defaults`, `properties`.

But `ConfigProviderDefinition.to_dict()` (in shinbot/core/config_provider/schema.py) returns:
`kind, id, display_name, description, config_version, fields, example_toml, owner_module, source_path, metadata`

The dashboard relies on `fields` to render config forms. FastAPI's response_model filtering will strip `fields` and everything else not in the model, breaking the dashboard.

**Fix**: Rewrite `ConfigProviderSummary` to match the actual `to_dict()` output. Use field names matching the camelCase/dict keys the route returns. Add `extra="allow"` to the model config so any future fields pass through.

## Bug 2 (P1) — config_providers.py:100: ConfigProviderDefaults returns empty

The route at line 100 returns `ok(bot.config_provider_registry.default_config(kind, provider_id))` which is a raw dict like `{"mainLlm": "gpt-4", ...}`.

But the response model `ConfigProviderDefaults` wraps it as `{"defaults": {...}}`. FastAPI treats the raw keys as extras and filters them, so the caller gets `{"defaults": {}}`.

**Fix**: Change the response model to `Envelope[dict[str, Any]]` (or remove the response_model entirely for this endpoint) so the raw dict passes through as-is. Alternatively, change the route to wrap the return as `{"defaults": raw_dict}` and keep the model — but that would break existing callers. Safest: use `Envelope[dict[str, Any]]`.

## Bug 3 (P2) — model_runtime.py:173: ExecutionData timestamps not nullable

`ExecutionData.firstTokenAt` and `ExecutionData.finishedAt` are declared as `str` (required, non-nullable).

But for non-completion calls (embedding, rerank, video), `first_token_at` is legitimately None. SQLite also defines `finished_at` as nullable. When such records exist, `/model-runtime/executions` will 500.

**Fix**: Change both fields to `str | None = None`.
