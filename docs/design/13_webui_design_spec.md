# ShinBot 技术规范：WebUI 架构与设计 (WebUI Design)

本文档定义了 ShinBot 管理后台的视觉规范、导航结构及各功能模块的实现细节。

---

## 1. 技术栈选型
- **框架**: Vue 3 (Composition API)
- **构建**: Vite + pnpm
- **UI 库**: Vuetify 3 (Material Design 风格)
- **状态管理**: Pinia
- **国际化**: Vue I18n
- **通信**: Axios + WebSockets (用于实时日志)

---

## 2. 核心布局规范
- **登录墙 (Login Wall)**: 强制鉴权，未登录用户无法访问任何内部路由。
- **主布局**: 
    - **左侧导航栏 (Sidebar)**: 支持折叠为图标模式。
    - **顶部工具栏 (Header)**: 显示当前实例状态、通知及用户菜单。
    - **主体区域 (Main)**: 右侧动态加载内容页面。

---

## 3. 导航模块规划

### 3.1 核心页面
- **欢迎 (Welcome)**: 引导式配置流。支持在导航栏关闭，可在系统设置恢复。
- **概览 (Dashboard)**: 系统运行指标、活跃会话、告警摘要。
- **机器人 (Instances)**: 
    - 卡片式布局管理接入端。
    - **编辑叠层 (Overlay)**: 配置适配器、默认插件组（黑白名单模式）、Agent 预设。

### 3.2 插件管理 (Plugins)
- **管理**: 
    - 卡片展示已安装插件。
    - **配置面板**: 根据插件提供的 `ConfigSchema` 自动生成动态表单。
- **市场**: 仓库源驱动的插件列表（规划中）。
- **编组**: 将常用插件打包为“组”，实现多机器人间的配置快速平移。

### 3.3 权限体系 (Permissions)
- **权限组管理**: 跨实例的成员绑定，不依赖于具体实例的开启状态。
- **指令管理**: 全局指令注册表查询与开关。
- **自定义规则**: 精细到 `instance:session` 维度的插件/Agent 触发规则。

### 3.4 监控中心 (Monitoring)
- **日志**: 虚拟滚动的高性能终端模拟器。
- **对话数据**: 会话历史上下文管理。
- **审计 (规划中)**: 成本分析（Token/费用）、模型追踪（Prompt 审计）、定时任务监控。

---

## 4. UI 交互原则
- **操作即反馈**: 所有的修改应通过 Toast 或 Snackbar 给予明确结果。
- **数据一致性**: 利用 Pinia 确保导航栏的状态与页面内容实时同步。
